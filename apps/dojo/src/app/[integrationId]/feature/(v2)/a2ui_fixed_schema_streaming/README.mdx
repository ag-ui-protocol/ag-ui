# A2UI Fixed Schema (Streaming)

## What This Demo Shows

Streaming variant of the fixed-schema A2UI pattern. Cards appear progressively as the LLM streams tool arguments.

1. **Middleware-driven streaming**: The Python tool is a plain `@tool` — streaming is configured entirely in the runtime's `streamingSurfaces`
2. **Progressive rendering**: Schema is emitted at tool start, data streams as the LLM generates each flight
3. **Cards appear one-by-one**: Each flight card renders as soon as its data is complete
4. **Same action handlers**: "Select" button works identically to the non-streaming variant
