The AG-UI .NET SDK is designed for .NET applications that already use `Microsoft.Extensions.AI`. It exposes AG-UI endpoints as ordinary `IChatClient` instances and adapts existing `IChatClient` implementations into AG-UI-compatible HTTP and Server-Sent Events endpoints.

## Who this serves

- .NET application developers who want to expose an existing `IChatClient` as an AG-UI agent endpoint.
- .NET client developers who want to consume an AG-UI endpoint through the `Microsoft.Extensions.AI` middleware stack.
- Microsoft Agent Framework builders who need to validate that AG-UI preserves messages, tools, state, interrupts, reasoning, and multimodal input across the .NET boundary.

## Planned package map

| Package | Reader goal | Draft SDK role |
| --- | --- | --- |
| `AGUI.Abstractions` | Share protocol types without choosing a transport | Defines events, messages, tools, run input, interrupts, capabilities, multimodal input, raw event preservation, and the source-generated JSON serializer context. |
| `AGUI.Client` | Consume an AG-UI endpoint from .NET | Wraps an AG-UI server as an `IChatClient` with HTTP/SSE transport and event-to-update conversion. |
| `AGUI.Hosting.AspNetCore` | Host an AG-UI endpoint from ASP.NET Core | Converts inbound `RunAgentInput` into `ChatRequestContext` and outbound `ChatResponseUpdate` streams into AG-UI events. |

## Work that can start before the SDK merges

- Audit terminology across the AG-UI docs so .NET pages use the same protocol names as the existing TypeScript and Python references.
- Draft hosting and client journeys around `IChatClient`, `ChatMessage`, `ChatOptions`, `ChatResponseUpdate`, `AIContent`, and `AIFunction`.
- Document forward compatibility expectations: mapped concepts become MEAI updates, while unmapped AG-UI concepts remain available through raw event representations.
- Keep Microsoft Agent Framework-specific mappings separate from SDK conformance guidance; Agent Framework consumes the .NET SDK and owns its framework-specific event mapping tests.
- Prepare cross-links from Microsoft Agent Framework and Microsoft Learn pages once the Microsoft docs owners confirm the preferred destination.
- Create a final-package checklist for NuGet names, target frameworks, sample paths, and conformance commands.
