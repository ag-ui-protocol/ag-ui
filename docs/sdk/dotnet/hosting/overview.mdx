You have an ASP.NET Core application with an `IChatClient`. The hosting package should let you expose that client as an AG-UI endpoint without rewriting your agent around wire-level event types.

## Reader journey

Start with an ASP.NET Core application that already registers an `IChatClient`. End with a POST endpoint that accepts AG-UI `RunAgentInput`, streams AG-UI events over SSE, and keeps `Microsoft.Extensions.AI` middleware in the path.

## Core concepts to explain

### Register AG-UI serialization

The server setup needs to register AG-UI JSON metadata so ASP.NET Core can serialize and deserialize the protocol types without reflection-heavy runtime discovery.

Docs should cover:

- Where `AddAGUI()` belongs in service registration.
- How `AGUIJsonSerializerContext` is added to the `JsonSerializerOptions` resolver chain.
- How interrupt request and response content types are registered for polymorphic `AIContent` deserialization.
- How source-generated JSON serialization supports trim and native AOT scenarios.

### Adapt inbound AG-UI requests

`ToChatRequestContext` is the bridge from AG-UI protocol input to the `Microsoft.Extensions.AI` model.

Docs should cover:

- Mapping `RunAgentInput.Messages` into `ChatMessage` values.
- Preserving thread, run, and message identity.
- Carrying client-declared tools into `ChatOptions`.
- Handling multimodal input as standard MEAI content.

### Stream outbound protocol events

`AsAGUIEventStreamAsync` converts `ChatResponseUpdate` items into AG-UI events.

Docs should cover:

- Lifecycle events for run start and finish.
- Text message start, content, and end events.
- Tool call and tool result event mapping.
- State snapshot, state delta, reasoning, custom, raw, and interrupt mapping through `AGUIStreamOptions`.
- Raw event preservation for AG-UI concepts that do not have a first-class `Microsoft.Extensions.AI` representation.

## Example sequence to finalize

The final page should include one complete ASP.NET Core sample that:

1. Registers `AddAGUI()`.
2. Registers an `IChatClient` through the standard MEAI builder pipeline.
3. Maps a POST endpoint.
4. Converts `RunAgentInput` to `ChatRequestContext`.
5. Streams `BaseEvent` values as SSE items.
